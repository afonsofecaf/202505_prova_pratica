🧪 Prova Prática Advanced Databases Turma 01 (22/05/2025) – Envio de CSV para Firebase (1,5 ponto)

Objetivo: Criar um script no Google Colab que leia um arquivo CSV e envie seus dados para o Firebase Firestore, respeitando uma estrutura específica de nomeação de collections e documentos. A prova irá avaliar, além do envio geral dos dados, a segregação automática das collections por tipo de dado.

📌 Instruções

  • Baixe automaticamente um arquivo CSV da UCI Machine Learning Repository.
  • Leia o conteúdo do arquivo linha a linha.
  • Crie uma collection no Firebase com o nome do arquivo (sem a extensão .csv).
  • Insira cada linha do CSV como um documento separado, usando IDs no padrão input001, input002, etc.
  • Regra obrigatória: Após criar a collection geral, crie collections adicionais automaticamente, segregando os dados conforme o tipo de cada coluna:
    • Identifique no DataFrame todos os tipos de dados presentes (ex: numérico, string, booleano).
    • Para cada tipo identificado, crie uma nova collection com nome no padrão nomearquivo_tipo (exemplo: iris_numerico, iris_string).
    • Cada collection segregada deve conter apenas as colunas correspondentes ao tipo de dado (exemplo: apenas as colunas numéricas na collection _numerico).
    • Mantenha o padrão de IDs inputNNN nas collections segregadas.
Exemplo de arquivo sugerido:
Dataset: Iris CSV
Nome esperado ao salvar: iris.csv

🧾 Requisitos

  • Script em Python no Google Colab.
  • Utilize o Firebase Admin SDK para envio dos dados.
  • Inclua prints de tela mostrando a collection geral e as collections segregadas criadas no Firestore no colab.
  • O notebook deve estar bem documentado e com instruções claras.

⚙️ Como fazer no Google Colab

  1. Configure o Firebase: gere a chave JSON e faça o upload no Colab.
  2. Instale o Firebase Admin SDK: !pip install firebase-admin
  3. Utilize pandas para ler o CSV: import pandas as pd
  4. Envie todos os dados para a collection geral, criando documentos com IDs no padrão solicitado.
  5. Automaticamente, separe os dados por tipo (numérico, string, booleano) e envie cada subconjunto para uma collection própria, conforme explicado acima.
  6. Compartilhe o link público do Colab no formulário abaixo.
⚠️ Atenção: certifique-se de que o link do Google Colab esteja com a opção "Qualquer pessoa com o link pode visualizar" ativada!
